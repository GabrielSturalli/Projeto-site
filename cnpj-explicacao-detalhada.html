Prospecção de Empresas via Dados Públicos da Receita Federal

Como os dados abertos do governo federal permitem identificar, filtrar e priorizar empresas recém-abertas — e como isso foi automatizado em Python.

Fonte dos dadosReceita Federal do Brasil
Registros processados~68,6 milhões de CNPJs
Produto finalListas Excel segmentadas
O problema

A janela de oportunidade que a maioria das empresas não enxerga

Toda empresa, no momento em que é registrada na Receita Federal, existe formalmente — mas ainda não tem conta bancária, maquininha de cartão, folha de pagamento, nem nenhuma outra relação financeira ou comercial estabelecida. Esse intervalo, que pode durar de alguns dias até dois ou três meses, representa a janela de aquisição mais valiosa do ciclo de vida de uma empresa cliente.

A dor central: negócios de aquisição costumam gastar boa parte do orçamento de marketing tentando convencer empresas que já possuem um fornecedor/parceiro a trocar. O custo de conversão é alto e a resistência é grande. Abordar uma empresa antes de ela ter qualquer vínculo estabelecido inverte essa equação.

O problema, até então, era que localizar essas empresas no momento certo exigia cruzar dados de fontes diferentes, processar dezenas de milhões de registros e aplicar filtros estratégicos por segmento, região e perfil financeiro — uma operação de engenharia de dados que a maioria das equipes comerciais não consegue fazer de forma ágil.

A solução

Dados públicos como inteligência competitiva

A Receita Federal disponibiliza publicamente, todos os meses, o cadastro completo de todas as pessoas jurídicas do Brasil. Esses arquivos são gratuitos, legais, e contêm informações sobre cada empresa registrada no país — incluindo data de abertura, atividade econômica, capital social e localização.

ArquivoO que contémVolume
Empresas*.zipRazão social, capital social, natureza jurídica, porte~68,6 milhões de registros
Estabelecimentos*.zipEndereço, CNAE, data de abertura, situação cadastral~60 milhões de registros
Cnaes.zipTabela de segmentos econômicos (1.359 categorias)Tabela auxiliar

O desafio técnico é que esses arquivos somam mais de 8 GB comprimidos e não possuem uma interface de consulta nativa. Para extrair valor deles, é necessário construir um pipeline de processamento eficiente que carregue os dicionários de referência em memória, filtre os registros por critérios estratégicos e entregue o resultado em formato utilizável.

Arquitetura técnica

Como o pipeline de dados foi construído

1

Carregamento dos dicionários em memória

O arquivo Empresas é carregado primeiro, construindo um dicionário Python com chave CNPJ_BASE (8 dígitos). Isso permite fazer lookups em O(1) durante o processamento principal, sem precisar fazer joins em disco — técnica essencial para processar dezenas de milhões de registros sem travar.

# Dicionário: cnpj_base → (razão social, capital social)
empresas[cnpj_base] = (razao, capital)
2

Streaming dos Estabelecimentos com filtros em cascata

Os arquivos de Estabelecimentos — os maiores, com até 2 GB cada — são lidos em streaming via csv.reader, sem carregar tudo na memória. A cada linha, os filtros são aplicados em ordem crescente de custo: os mais rápidos (situação cadastral, tipo matriz) são verificados primeiro, descartando a maioria dos registros antes de chegar ao filtro de data ou CNAE.

for row in reader:
    # Filtros rápidos primeiro — descartam ~90% dos registros
    if row[5].strip() != "02": continue  # apenas ativas
    if row[3].strip() != "1":  continue  # apenas matrizes

    # Filtros estratégicos
    if data_raw < DATA_MINIMA:    continue  # data de abertura
    if cnae_cod not in CNAES_ALVO: continue  # segmento
    if capital < CAPITAL_MINIMO:  continue  # saúde financeira
3

Enriquecimento e formatação

Para cada registro que passa pelos filtros, o pipeline enriquece os dados: formata o CNPJ com pontuação, converte a data de AAAAMMDD para DD/MM/AAAA, e traduz o código CNAE para a descrição do segmento. Caracteres ilegais para Excel são sanitizados com regex.

def format_cnpj(base, ordem, dv):
    b = base.zfill(8)
    return f"{b[:2]}.{b[2:5]}.{b[5:8]}/{ordem.zfill(4)}-{dv.zfill(2)}"

ILLEGAL_CHARS = re.compile(r"[\x00-\x08\x0b\x0c\x0e-\x1f]")
def clean(v): return ILLEGAL_CHARS.sub("", str(v)) if v else ""
4

Geração do Excel com openpyxl (write-only)

O Excel é gerado usando o modo write_only do openpyxl, que escreve linhas diretamente no arquivo sem manter a planilha em memória — crítico para arquivos com centenas de milhares de linhas. Cada célula recebe formatação individual: fonte, borda, alinhamento e cor de fundo diferenciada por mês de abertura.

# Cores por mês de abertura — destaque visual de recência
FILL_MES_1 = PatternFill("solid", start_color="FFFF00")
FILL_MES_2 = PatternFill("solid", start_color="C6EFCE")

# Salva a cada N linhas para não estourar o limite do Excel
if row_count >= CHUNK_SIZE:
    wb.save(path)
    wb, ws = create_workbook()  # abre nova parte

Valor de negócio

Por que empresas recém-abertas são o melhor momento para abordagem

~60
dias é a janela média antes de a empresa estabelecer vínculos comerciais/bancários principais
0
custo de switching — a empresa ainda não tem nada a "trocar"

Empresa nova tem urgência real: precisa se organizar operacionalmente, emitir nota fiscal, receber pagamentos de clientes, pagar fornecedores. Essa urgência cria receptividade natural que não existe em empresas maduras já com fornecedores e parceiros estabelecidos.

A combinação de segmento certo + timing certo + canal certo cria uma proposta de abordagem com custo de aquisição muito inferior ao marketing tradicional.

Diferencial competitivo: os dados da Receita Federal são públicos e qualquer empresa pode baixá-los. O valor não está no dado — está em saber quais perguntas fazer: quais segmentos têm maior propensão de conversão, qual nível de capital social indica viabilidade do negócio, e qual janela de abertura cria a urgência certa de contato.
